# 探索智能体与观察智能体交互过程图

## 用途

展示论文3.3.1节中探索智能体（Exploration Agent）与观察智能体（Observation Agent）的协同诊断交互过程，包括三次SOP跳转：第一次遵循SOP跳转，第二、三次逃逸创建新节点。

## 设计说明

### 布局结构
- **三列布局**：每列展示一次SOP跳转过程
- **左侧**：SOP树（从上到下延伸，显示节点和边，红色节点表示当前SOP位置）
- **右侧**：
  - 顶部：当前SOP节点信息
  - 下方：六条消息的对话过程（类似微信对话风格）

### 对话消息顺序（每列6条）
1. 探索智能体：提问（紫色，靠右）
2. 观察智能体：思考（蓝色，靠左）
3. 观察智能体：调用工具（橙色，靠左）
4. 观察智能体：返回结果（蓝色，靠左）
5. 探索智能体：思考（紫色，靠右）
6. 探索智能体：做出决定（绿色=沿SOP跳转 / 橙色=逃逸）

### 三列差异
- 第一列：✅ 沿SOP边跳转
- 第二列：🚀 逃逸：创建新节点
- 第三列：🚀 逃逸：创建新节点

### 颜色方案
| 颜色 | 用途 |
|------|------|
| 紫色 #9B59B6 | 探索智能体消息 |
| 蓝色 #3498DB | 观察智能体消息 |
| 橙色 #D35400 | 工具调用 |
| 绿色 #27AE60 | 沿SOP跳转决策 |
| 橙色 #E67E22 | 逃逸决策 |
| 红色 #E74C3C | 当前SOP节点 |
| 灰色 #BDC3C7 | SOP树普通节点 |
